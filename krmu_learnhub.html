Step 1 — Hello World & Program Structure
Concept · Code · Quiz
▶
Every C++ program starts with main(). The compiler reads your code top to bottom. Include #include <iostream> to print output.
C++ · hello.cpp
#include <iostream> // gives us cout and cin using namespace std; // no need to write std:: every time int main() { cout << "Hello, KRMU!" << endl; return 0; // 0 means program ran successfully }
💡 Memory trick: cout = "Character OUT" → sends text to screen. cin = "Character IN" → reads from keyboard. Arrows show data direction: << out, >> in.

Step 2 — Variables & Data Types
Concept · Code · Quiz
▶
A variable is a named storage box. Declare type first, then name. C++ has 5 main types you'll use every day.
C++ · datatypes.cpp
int age = 20; // whole number: -5, 0, 42 double cgpa = 8.5; // decimal: 3.14, 9.99 char grade = 'A'; // ONE character, use ' ' bool passed = true; // only true or false string name = "Rahul"; // text, use " " cout << name << " CGPA: " << cgpa << endl;
⚠️ Common mistake: Don't use ' single quotes for strings — that's for single characters only. Use " double quotes for text!

Step 3 — Control Flow (if/else & Loops)
Code · Practice Challenge
▶
Control flow = deciding what runs and how many times. Master if/else for decisions, for and while loops for repetition.
C++ · grading.cpp
int marks = 78; if (marks >= 90) cout << "Grade A — Excellent!"; else if (marks >= 70) cout << "Grade B — Good job!"; // ← this runs else if (marks >= 50) cout << "Grade C — Keep going!"; else cout << "Grade F — Revise!"; // for loop: runs exactly 5 times for (int i = 1; i <= 5; i++) { cout << "Count: " << i << endl; } // while loop: runs as long as condition is true int n = 1; while (n <= 5) { cout << n*n << " "; n++; } // prints: 1 4 9 16 25

Concept · Code · Quiz
▶
A function is a reusable block of code. Write once, call many times. Functions have: return type, name, parameters, and a body.

Step 5 — OOP: Classes & Objects
Advanced · Code · Project
▶
OOP = model real-world things as code objects. A class is the blueprint (e.g. "Student"), an object is a real instance (e.g. "Rahul"). Four pillars: Encapsulation, Inheritance, Polymorphism, Abstraction.
C++ · Student class
class Student { private: string name; double cgpa; // private: only class can access public: // Constructor — runs when object is created Student(string n, double c) : name(n), cgpa(c) {} void display() { cout << name << " — CGPA: " << cgpa << endl; } bool isTopper() { return cgpa >= 9.0; } }; int main() { Student s1("Rahul", 9.2); // create object s1.display(); if (s1.isTopper()) cout << "🏆 Topper!"; }

Present Perfect vs Simple Past

Simple Past = completed action at a specific time. Present Perfect = past action with present relevance.

❌ "I have submitted the assignment yesterday."
✅ "I submitted the assignment yesterday." (Simple Past — specific time)
✅ "I have submitted the assignment." (Present Perfect — relevant now)

Conditional Sentences

First conditional = real/possible. Second conditional = hypothetical/imaginary.

✅ "If I study hard, I will pass the exam." (First — possible)
✅ "If I were a data scientist, I would work at Google." (Second — hypothetical)

Technical Writing: Active vs Passive

In technical writing, active voice is clearer and more direct than passive.

❌ "The algorithm was implemented by the team." (Passive)
✅ "The team implemented the algorithm." (Active — clearer!)

Step 1 — OS Concepts: Processes & Scheduling
Concept · Quiz
▶
An Operating System manages hardware and software. Key concepts: Process (running program), Thread (sub-process), Scheduling (who runs when), Memory Management (RAM allocation).
💡 Process States: New → Ready → Running → Waiting → Terminated. The OS scheduler decides which process gets CPU time using algorithms like FCFS, Round Robin, SJF.
